# GitHub Pages × marked.js で自分専用ブログを作った理由

## なぜ WordPress を使わなかったのか
ブログを始める際、最も一般的な選択肢は WordPress だろう。しかし、自分はあえて「静的サイト」としての自作を選択した。理由は、単に記事を書くだけでなく、Webの仕組みそのものを自分の手でコントロールしたかったからだ。

## 1. 究極の表示速度へのこだわり
WordPressはデータベースとの通信が発生するため、どうしても動作が重くなりがちだ。今回構築したシステムは、HTML/CSS/JSのみで構成されている。

* **サーバーレス**: GitHub Pages を利用することで、爆速で配信される。
* **軽量化**: 余計なプラグインを一切排除し、読み込み時間を極限まで短縮した。

## 2. menu.js によるスマートな共通化
複数のページを管理する上で、ヘッダーの更新は手間がかかる。自分は `menu.js` という共通スクリプトを作成し、一行書き換えるだけで全ページのメニューが更新される仕組みを構築した。

## 3. marked.js による Markdown 運用
記事の執筆には `marked.js` を採用した。これにより、VS Codeなどのエディタで書いた `.md` ファイルを `posts/` フォルダに置くだけで、自動的にレンダリングされる。HTMLタグを意識せず、思考のスピードで記事を書ける環境が整った。

## まとめ：自作こそが最大の学び
既存のサービスを使えば数分でブログは作れる。しかし、自分でコードを書き、解決していく過程で得られる知識は何物にも代えがたい。このサイト自体が、自分の「制作実績」の一つである。
